## Critical Rules (quick reference)
- JMB/MC buildings are ALWAYS free
- Face IS identity — no manual clock-in, no name dropdown
- Face enrollment is merged into the Edit staff modal photo upload (no standalone Enroll button) — D-0382
- Technician = role:staff, position:Technician (NOT contractor_worker)
- Hardware Store has NO inventory (Telegram orders only)
- Attendance: all reads via Postgres RPCs, never PostgREST
- RPC caller rule: when changing RPC signatures, update ALL callers + NOTIFY pgrst 'reload schema' in same commit
- Auth gate rule: PWA routes (/api/app/*) use getSession(), BM portal uses requirePermission() — never swap
- Portal routing: lib/auth/get-portal-path.ts is the ONLY source of truth. middleware.ts uses it directly. NEVER change without human approval (D-0370/D-0379)
- Org admin auth: email + password via /vendor-login. NOT Google OAuth (D-0386)
- PMC model: PMC org owns staff in Malaysia — employs BM/admin/accounts, deploys to buildings. JMB can hire own staff directly. Both models coexist.
- Crons: migrated to cron-job.org. vercel.json crons=[]. 10 schedules. See docs/startup/cron-external-setup.md. All routes: Bearer CRON_SECRET, ?job= param for single-task testing.
- Types file regenerated 2026-04-10 (V20, migration 070 — migration 071 is 6 RPCs, no new columns, no regen needed)
- AI model strings: ROUTINE = `deepseek-chat`, SENSITIVE = `claude-sonnet-4-20250514` (never use aliases)
- **MO roles:** Only Admin + Technician. No "staff" role in UI. DB keeps role:'staff'+position:'Technician' pattern unchanged.
- **Service worker:** /app in PROTECTED_PREFIXES, CACHE_NAME=minilab-v3. Bump cache version on any PWA code change.
